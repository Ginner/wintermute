# AGENTS.md — hosts/

Each subdirectory is one machine. The directory name is the hostname.

## Directory structure per host

```
hosts/<HOSTNAME>/
  configuration.nix         ← NixOS system config for this host
  hardware-configuration.nix ← generated by nixos-generate-config; hardware-specific
  home.nix                  ← HM entrypoint for this host (wired in via configuration.nix)
  .secrets/secrets.nix      ← agenix public key declarations for this host (optional)
```

## What each file contains

**`configuration.nix`**:
- `imports`: hardware-configuration.nix, `../../nixosModules`, `../../users/<username>`, and any extra flake NixOS modules (e.g. xremap-flake)
- `networking.hostName`
- `userGlobals.username = "<username>"`
- One bundle enable: `myModules.laptop.enable = true` (or desktop/server)
- Host-specific module overrides: `myModules.services.*.enable = true/false`, option values
- `home-manager.extraSpecialArgs = { inherit inputs; }`
- `home-manager.users."<username>" = import ./home.nix`
- `system.stateVersion` — set at install time, do not change

**`hardware-configuration.nix`**:
- Generated by `nixos-generate-config --root /mnt`
- Contains boot modules, filesystem UUIDs, swap devices, CPU microcode
- Only edit for filesystem/boot specifics that `nixos-generate-config` gets wrong
- Do not put service or program config here

**`home.nix`**:
- `imports`: `../../homeManagerModules`, `../../users/<username>/home.nix`, plus any HM flake modules (nixvim, ags, walker)
- One HM bundle enable: `myHomeModules.laptop.enable = true`
- Host-specific HM overrides: monitor profiles (`services.kanshi.settings`), wallpaper overrides, per-host package additions
- `home.stateVersion` — set at install time, do not change

**`.secrets/secrets.nix`**:
- Declares which age public keys can decrypt which secrets
- Contains only public key variables and secret file declarations
- Required by agenix; no private keys here

## Patterns observed

- All host names are uppercase
- The host's `configuration.nix` imports `../../users/<username>` (NixOS side) and the host's `home.nix` imports `../../users/<username>/home.nix` (HM side)
- Host-specific kanshi monitor profiles are set directly with `services.kanshi.settings` in `home.nix` rather than via the module option — this is the correct place for per-host display configuration
- Wallpaper is overridden in `home.nix` via `myHomeModules.guiPrograms.stylix.image`
